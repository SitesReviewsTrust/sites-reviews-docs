# Contributing to sites-reviews-docs

Thanks for helping improve the Sites.Reviews documentation hub. This repo is the cross-linking center for the whole ecosystem, so keeping it clear and accurate matters.

## What belongs here

- Fixes to typos, wording, broken links, and formatting.
- Clarifications and small additions to existing docs (`README.md`, `docs/*`).
- Keeping the **ecosystem table** and **cross-link block** in sync with reality.

## What belongs elsewhere

- **Browser extension** issues/PRs → [sites-reviews-extension](https://github.com/SitesReviewsTrust/sites-reviews-extension)
- **MCP server** issues/PRs → [sites-reviews-mcp](https://github.com/SitesReviewsTrust/sites-reviews-mcp)
- **API & widgets** issues/PRs → [sites-reviews-api](https://github.com/SitesReviewsTrust/sites-reviews-api)

## Ground rules

1. **Stay factual.** Don't add metrics, endpoints, or features that don't exist. If something is in beta or not yet available, say so plainly.
2. **Mark inferences.** If you describe how something *generally* works (rather than a published behaviour), label it as such — see [docs/trust-score.md](./docs/trust-score.md) for the style.
3. **Keep the cross-link block verbatim.** The shared `🔗 Sites.Reviews ecosystem` block is intentionally identical across files. If it changes, update it everywhere.
4. **English first** (this is a public, global, AI-discoverable repo); a short Russian section is welcome.

## How to contribute

1. Fork the repo and create a branch.
2. Make your change.
3. Open a pull request with a short description of what and why.
